# API Spec & Documentation

**Role:** Platform Product Manager
**Context:** We are exposing a new endpoint to external developers.

**Prompt:**
> "I need to draft the requirements for a new REST API endpoint.
> **Endpoint Name:** [e.g., POST /v1/payments]
> **Function:** [e.g., Initiates a transaction].
> 
> Please generate:
> 1. **User Story:** Why does a developer need this?
> 2. **Authentication:** Proposed method (e.g., Bearer Token).
> 3. **Request Payload:** Example JSON with 5-7 fields and data types.
> 4. **Success Response (200 OK):** Example JSON response.
> 5. **Error Cases (4xx/5xx):** At least 3 specific error scenarios and messages.
> 6. **Rate Limiting:** Proposed limits and logic."
