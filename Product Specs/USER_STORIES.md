# User Story & Acceptance Criteria

**Role:** Agile Product Manager
**Context:** I need to prepare a backlog for an upcoming sprint.

**Prompt:**
> "I have the following feature requirement: [INSERT REQUIREMENT].
> 
> Please break this down into 3-5 individual User Stories using the format:
> 'As a [USER TYPE], I want to [ACTION], so that [VALUE].'
> 
> For each user story, provide:
> 1. **Acceptance Criteria (AC):** Bulleted list of must-haves (Given/When/Then or simple bullets).
> 2. **Technical Implementation Notes:** Brief considerations for developers.
> 3. **Design Considerations:** UI/UX requirements for the designer."
