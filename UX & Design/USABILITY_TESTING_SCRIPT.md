# 🧪 Usability Testing Script

**Goal:** Create a structured usability testing script to validate assumptions, uncover pain points, and ensure a smooth user experience.

---

### 📝 The Prompt

```text
Act as a Lead UX Researcher. We are preparing to conduct usability testing for:
"[INSERT FEATURE - e.g., 'A new feature that allows users to seamlessly switch between multiple accounts without logging out']"

We want to test this feature with:
"[INSERT TARGET USER - e.g., 'Power users or administrators managing multiple client profiles']"

The primary goal of this testing is to learn:
"[INSERT GOAL - e.g., 'If the account switcher is easily locatable and whether users understand which account they are currently operating in']"

Please generate a structured usability testing script. Include the following sections:
1. **Introduction:** A warm greeting, context-setting, and gaining consent to record.
2. **Warm-up Questions (2-3):** Broad questions to get the user comfortable and learn about their current workflow.
3. **Core Tasks (3-4):** Specific, scenario-based tasks for the user to complete without leading them (e.g., "Show me how you would...").
4. **Follow-up Questions:** Questions to ask after each task to probe deeper into their experience or any struggles they had.
5. **Debrief/Wrap-up:** Final thoughts, open feedback, and thanking the user.

Ensure the tone is neutral, welcoming, and strictly focused on testing the product, not the user.
```

---

### 💡 Pro-Tip
Never ask leading questions during a usability test (e.g., "Was it easy to find the account switcher?"). Instead, use task-based prompts (e.g., "Imagine you need to update settings for Client B. Show me what you would click."). This script helps enforce that best practice.
