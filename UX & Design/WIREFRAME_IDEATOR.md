# 🎨 Wireframe Ideator

**Goal:** Generate low-fidelity structural ideas and unblock the design process for a new feature. Use this to provide a starting point for discussions with your UX team.

---

### 📝 The Prompt

```text
Act as a Principal Product Designer. I need to conceptualize the wireframe structure for a new feature:
"[INSERT FEATURE DESCRIPTION - e.g., 'A dashboard widgets configuration panel for returning managers']"

The primary goal of this feature is: "[INSERT GOAL - e.g., 'Allow managers to quickly select and arrange their top 3 KPIs upon login']"
The target audience is: "[INSERT AUDIENCE - e.g., 'Busy sales managers who need at-a-glance metrics']"

Please provide a detailed text-based wireframe layout. Include:
1. **Overall Page Structure:** (e.g., Header, Sidebar, Main Content Area)
2. **Key Components:** Break down each section into specific UI elements (e.g., buttons, dropdowns, lists) and their visual hierarchy.
3. **Core Interactions:** Describe what happens when a user clicks/taps the primary calls to action.
4. **Information Architecture:** What information needs to be most prominent vs. secondary?
5. **Design Rationale:** Briefly explain why this structure serves the primary goal and target audience.
```

---

### 💡 Pro-Tip
Use the output not as a final spec, but as a collaborative tool. Take this text layout into a whiteboard session with your designer and use it to sketch out v1 together.
